# Quick Task: Fix MD5 Hashing Bottleneck

## Goal
Replace the `aiofiles` chunk-by-chunk MD5 computation (8192-byte chunks, CPU work on event loop) with `asyncio.to_thread` wrapping a synchronous read+hash loop using 1MB chunks.

## Root Cause
`_calculate_md5` uses `aiofiles` which offloads disk reads to a thread pool but returns each chunk to the event loop where `hasher.update(chunk)` runs synchronously. With 8192-byte chunks and ~50 concurrent coroutines, the event loop is constantly receiving tiny chunks and doing CPU work between them — resulting in ~1.3 files/second.

## Fix

### Task 1: Replace `_calculate_md5` with thread-based implementation

**File:** `aws_copier/core/file_listener.py`

**Action:**
1. Add `_calculate_md5_sync` — a regular (non-async) method that opens the file, reads in 1MB chunks, and returns the hex digest
2. Replace `_calculate_md5` body: instead of aiofiles loop, call `await asyncio.to_thread(self._calculate_md5_sync, file_path)`
3. Remove unused `aiofiles` import if it's only used for MD5 (keep if used elsewhere)

**Verify:** `uv run pytest tests/ -x -q` passes

**Done:** `_calculate_md5` body is a single `asyncio.to_thread` call; no aiofiles in the hot path
