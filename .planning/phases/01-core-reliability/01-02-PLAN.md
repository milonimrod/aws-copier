<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `pyproject.toml` entrypoint → installed CLI | Any user who `pip install aws-copier` invokes the registered console script. A broken entrypoint surfaces as a user-facing error, not a security incident, but correctness still matters for supply-chain hygiene. |
| Runtime dependency list | Dependencies declared as runtime are shipped to every user who installs the package, expanding the attack surface. `ruff` is a dev tool — bundling it into runtime installs forces users to pull ~40 MB of linter source they never run. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Denial of Service (user-facing) | Broken `aws-copier` entrypoint | mitigate | Fix entrypoint to `main:sync_main`. Acceptance criterion: `uv run aws-copier` no longer raises `ModuleNotFoundError: simple_main`. |
| T-02-02 | Supply-chain bloat / indirect vulnerability exposure | `ruff` and `python-dotenv` shipped as runtime deps | mitigate | Move both to `[dependency-groups].dev`. Acceptance criterion: `grep` inside `[project].dependencies` returns 0 hits for either package. Reduces the transitive runtime dependency surface. |
| T-02-03 | Information Disclosure | `discovered_files_folder` write to `$HOME/.aws-copier/discovered` | accept | The field is dead (no code reads it). Removing it closes a latent risk that future code might write discovered-file manifests to disk without ignore-rule filtering. Disposition "accept-now-mitigated-by-removal". |
| T-02-04 | Tampering (config backward-compat) | Old user YAML files containing `discovered_files_folder` | mitigate | `SimpleConfig.__init__(**kwargs)` signature accepts and ignores unknown keys — tested by `test_legacy_config_with_discovered_files_folder_ignored`. No user config file breaks. |
</threat_model>
