<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Local YAML config file → SimpleConfig | User-owned config file content is parsed; trust assumed (single-user personal tool, file is local-only) |
| SimpleConfig → S3Manager (downstream Plan 03) | `use_credential_chain` flag controls whether explicit creds are passed to aiobotocore; mis-detection could either leak creds in logs or cause auth failure |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | SimpleConfig.to_dict / save_to_yaml | mitigate | Derived fields `use_credential_chain` and `credential_source` are NEVER added to `to_dict()` or `save_to_yaml()` (verified by tests `test_to_dict_excludes_derived_fields` and `test_save_yaml_excludes_derived_fields`). Prevents accidental write-back of credential-source labels into YAML. |
| T-02-02 | Spoofing | SimpleConfig credential detection (D-09) | mitigate | Detection uses presence-based check (`not raw_key or not raw_secret`) — never compares against placeholder strings. Test `test_credential_chain_when_key_empty_string` proves empty string also triggers chain fallback so a half-filled config does not silently authenticate with empty credentials at the AWS layer. |
| T-02-03 | Tampering | uv.lock | accept | uv.lock is committed and reviewed via PR; tampering is a project-trust assumption, not a runtime threat. ASVS L1 does not require lockfile signing for personal tools. |
| T-02-04 | Information Disclosure | pathspec dependency | accept | pathspec is a widely-vetted library (used by pip, black, mypy themselves); vulnerability footprint is the same as those tools. No additional mitigation beyond version pinning. |
| T-02-05 | Repudiation | Credential source logging (D-10) | mitigate | `credential_source` attribute set deterministically based on key/secret presence; logged once at startup by S3Manager (Plan 03) for audit trail. |
</threat_model>
