       ```python
       def _load_backupignore_spec(self, folder_path: Path, watch_root: Path) -> PathSpec:
           """Accumulate .backupignore patterns from watch_root down to folder_path.

           D-07: Patterns cascade into subdirectories — a rule in /photos/.backupignore
                 applies to /photos/2024/ as well.
           D-08: Child directory rules ADD to (not replace) ancestor rules.
                 Evaluated in path order: root → parent → child.
           Pitfall 4: Caller must pass the path relative to watch_root (with forward
                      slashes) to spec.match_file; absolute paths silently fail.

           Args:
               folder_path: Current directory being scanned
               watch_root: Root watch folder (ancestor boundary)

           Returns:
               PathSpec compiled from all applicable .backupignore files. If none
               exist or all reads fail, the returned spec matches no files.
           """
           all_patterns: List[str] = []
           try:
               parts = folder_path.relative_to(watch_root).parts
           except ValueError:
               parts = ()
           current = watch_root
           # Walk root → ... → folder_path, collecting .backupignore files in path order
           for part in ("",) + parts:
               if part:
                   current = current / part
               ignore_file = current / ".backupignore"
               if ignore_file.exists():
                   try:
                       lines = ignore_file.read_text(encoding="utf-8").splitlines()
                       all_patterns.extend(lines)
                   except Exception as e:
                       logger.warning(f"Could not read {ignore_file}: {e}")
           return PathSpec.from_lines("gitignore", all_patterns)
       ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User-supplied `.backupignore` content → pathspec | The user (single trust principal) writes pattern files; pathspec parses them. Malformed patterns are silently ignored by pathspec. |
| File system path traversal (`folder_path.relative_to(watch_root)`) | An attacker who can control which folder is scanned could bypass the cascade by passing a folder outside watch_root; the helper guards this with `try/except ValueError → parts = ()`. |
| main.py startup → AWS S3 (lifecycle API) | Single network call at startup. Failures are caught by `ensure_lifecycle_rule` itself (D-11). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-23 | Tampering | Malformed .backupignore | mitigate | The `_load_backupignore_spec` helper wraps the read in `try/except Exception as e` that logs a warning and continues. A malformed file produces an empty pattern contribution from that level. Verified by `test_unreadable_backupignore_does_not_crash`. |
| T-02-24 | Information Disclosure | Pitfall 4 — absolute path leakage | mitigate | The helper uses `folder_path.relative_to(watch_root)` and the caller passes `str(relative).replace("\\", "/")` to `match_file`. Absolute paths never reach `match_file`. Verified by `test_directory_scoped_pattern_matches_via_relative_path` (the directory-scoped pattern `raw/*.jpg` only matches if relative-path normalisation works). |
| T-02-25 | Denial of Service | Pathological regex from .backupignore | accept | pathspec compiles gitignore patterns to regex internally. A user with write access to `.backupignore` could in principle craft patterns that take exponential time, but this requires write access to the watched folder — same trust principal as the daemon owner. Single-user threat model. |
| T-02-26 | Elevation of Privilege | .backupignore overriding global IGNORE_RULES | mitigate | The `.backupignore` filter runs AFTER `IGNORE_RULES.should_ignore_file` in `_scan_current_files`. A malicious `.backupignore` cannot RE-INCLUDE files that the global ignore rules already exclude (e.g., `.env`, SSH keys). The rule order is enforced: global deny first, then per-folder add-on filtering. Acceptance criterion grep confirms `ignored_files` increments live in TWO places — both excluding, never re-including. |
| T-02-27 | Information Disclosure | Credential source log (D-10) | accept | The string `"config.yaml"` or `"provider chain (env / ~/.aws/credentials / IAM)"` is metadata — neither path is a secret on its own. The actual credential VALUE is never logged. ASVS L1 acceptable. |
| T-02-28 | Denial of Service | ensure_lifecycle_rule blocking startup | mitigate | Plan 03 already guarantees `ensure_lifecycle_rule` never raises. This plan adds the call but does not change its semantics. Even an AWS API hang is bounded only by the underlying aiobotocore/botocore default timeout; if that becomes a problem, future work could wrap the call in `asyncio.wait_for`. Acceptable for v2 (D-11 explicit acceptance of best-effort semantics). |
| T-02-29 | Spoofing | Pitfall 4 path normalisation on Windows | mitigate | `relative_str = str(relative).replace("\\", "/")` ensures Windows backslash paths are normalised before `match_file`. Without this, gitignore patterns silently fail to match on Windows. Test `test_directory_scoped_pattern_matches_via_relative_path` covers this. |
