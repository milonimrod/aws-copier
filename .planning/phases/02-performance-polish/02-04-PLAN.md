<objective>
Implement PERF-04: 2-second per-path debounce on file events in `FileChangeHandler`. Rapid successive events for the same path collapse to a single `_process_changed_file` invocation, eliminating redundant scans on atomic-save editor patterns.

Purpose: Editors like Vim, Emacs, and many IDEs perform "atomic save" by writing to a temp file then renaming. This produces multiple events for what the user perceives as one save. Without debounce, each event triggers a full folder scan, wasting CPU and network. The 2-second window collapses these bursts.

Output: New `_schedule_debounced` and `_debounced_process` coroutines on `FileChangeHandler`; new `_debounce_tasks: Dict[str, asyncio.Task]` instance dict; new `cancel_all_pending()` method called from `FolderWatcher.stop()`; updated `on_any_event` that schedules the debounced wrapper instead of `_process_changed_file` directly. Behaviour-proving tests verify rapid events collapse, distinct paths debounce independently, cancellation does not log errors, and shutdown cleans up pending tasks.
</objective>

       ```python
       async def _schedule_debounced(self, file_path: Path, event_type: str) -> None:
           """Schedule a debounced _process_changed_file call for file_path.

           D-06: Cancels any pending debounce task for this path and creates a fresh
           2-second timer. Dict access is safe — this coroutine always runs in the
           asyncio event loop thread (scheduled via run_coroutine_threadsafe).

           Args:
               file_path: Path that triggered the file system event
               event_type: watchdog event type string (created, modified, etc.)
           """
           key = str(file_path)
           existing = self._debounce_tasks.get(key)
           if existing is not None and not existing.done():
               existing.cancel()
           task = asyncio.create_task(
               self._debounced_process(file_path, event_type),
               name=f"debounce-{file_path.name}",
           )
           self._debounce_tasks[key] = task

       async def _debounced_process(self, file_path: Path, event_type: str) -> None:
           """Sleep 2 seconds then delegate to _process_changed_file.

           CancelledError is caught and silenced — it means a newer event superseded
           this one, which is normal behaviour, not an error.

           Args:
               file_path: Path to process after debounce window elapses
               event_type: watchdog event type string
           """
           try:
               await asyncio.sleep(2)
               await self._process_changed_file(file_path, event_type)
           except asyncio.CancelledError:
               # Superseded by a newer event for the same path — normal, not an error.
               pass
       ```

    4. Add a `cancel_all_pending` method on `FileChangeHandler` (regular, non-async — it just cancels):
       ```python
       def cancel_all_pending(self) -> None:
           """Cancel all pending debounce tasks (Pitfall 2 guard).

           Called from FolderWatcher.stop() before observer.stop() to prevent
           "RuntimeError: Event loop is closed" from tasks firing after shutdown.

           Returns:
               None
           """
           for task in list(self._debounce_tasks.values()):
               if not task.done():
                   task.cancel()
           self._debounce_tasks.clear()
       ```

    5. Update `FolderWatcher.stop` (lines 151-165) to call `cancel_all_pending` on every handler BEFORE `self.observer.stop()`:
       ```python
       async def stop(self) -> None:
           """Stop watching folders."""
           if not self.running:
               return

           logger.info("Stopping folder watcher")

           # PERF-04 / Pitfall 2: cancel pending debounce tasks before stopping the
           # observer so they cannot fire after the event loop starts shutting down.
           for handler in self.handlers.values():
               handler.cancel_all_pending()

           # Stop the observer
           self.observer.stop()
           self.observer.join(timeout=5)

           self.running = False
           self.handlers.clear()

           logger.info("Folder watcher stopped")
       ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| watchdog OS thread → asyncio event loop | Cross-thread coroutine scheduling via `run_coroutine_threadsafe` is the existing trust boundary; debounce dict is accessed only from the asyncio side |
| File system events → debounce dict | A burst of events from a single editor save flows through this dict; an attacker generating events could grow the dict, but only with paths inside watched folders (single-user trust) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-18 | Denial of Service | _debounce_tasks unbounded growth | accept | The dict is keyed by file path; in normal use it grows only as fast as the user creates files. After each 2-second window fires, completed tasks remain in the dict until overwritten by a new event for the same path. The space is bounded by the count of distinct files touched within a short window — not unbounded in practice. ASVS L1 not applicable to single-user local daemons. |
| T-02-19 | Race Condition | _debounce_tasks concurrent access | mitigate | Per-research: the dict is accessed only from the asyncio event loop thread (`_schedule_debounced` runs after `run_coroutine_threadsafe` schedules it on `self.event_loop`). No GIL races possible because all mutations happen serially in the same coroutine run. `cancel_all_pending` runs from `FolderWatcher.stop` which is also an asyncio coroutine on the same loop. |
| T-02-20 | Information Disclosure | Task name `f"debounce-{file_path.name}"` | mitigate | Task name uses `file_path.name` (basename) only — no full paths leak into task names visible to `asyncio.all_tasks()`. Filenames are already known to anyone with process visibility. |
| T-02-21 | Denial of Service | Pitfall 2: post-shutdown CancelledError | mitigate | `cancel_all_pending` is called from `FolderWatcher.stop` BEFORE `observer.stop()` and BEFORE the event loop shuts down. Pending tasks are cancelled cleanly; their `CancelledError` is swallowed inside `_debounced_process`. Verified by `test_cancelled_error_silenced`. |
| T-02-22 | Tampering | Debounce window value (2 seconds) | accept | The window is locked by D-06 to 2 seconds. A user with code-edit access could change it; that is outside the runtime threat model. |
</threat_model>
