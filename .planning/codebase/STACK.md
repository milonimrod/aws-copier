# Technology Stack

**Analysis Date:** 2026-04-24

## Languages

**Primary:**
- Python 3.11 - All application code (pinned via `.python-version`)

**Secondary:**
- YAML - Configuration (`config.yaml`, `pyproject.toml`)

## Runtime

**Environment:**
- Python 3.11 (pinned in `.python-version`)
- Requires Python >=3.9 per `pyproject.toml`

**Package Manager:**
- `uv` - Dependency management and virtual environment
- Lockfile: `uv.lock` (present and committed)

## Frameworks

**Core:**
- `asyncio` (stdlib) - Async event loop powering all I/O operations
- None (no web framework — this is a CLI/daemon application)

**GUI:**
- `tkinter` (stdlib) - Desktop GUI in `aws_copier/ui/simple_gui.py`; ships with Python, no extra install

**Testing:**
- `pytest` >=8.4.1 - Test runner
- `pytest-asyncio` >=1.1.0 - Async test support (`asyncio_mode = "auto"` in `pyproject.toml`)
- `pytest-cov` >=6.2.1 - Coverage reporting (minimum 30% enforced)
- `moto[s3]` >=4.0.0 - AWS S3 mock for unit tests

**Build:**
- `hatchling` - Build backend (`[build-system]` in `pyproject.toml`)

## Key Dependencies

**Critical:**
- `aiobotocore` >=2.24.1 - Truly async AWS SDK wrapper around botocore; used in `aws_copier/core/s3_manager.py` for all S3 operations
- `watchdog` >=3.0.0 - Cross-platform file system event monitoring; used in `aws_copier/core/folder_watcher.py` via `Observer` and `FileSystemEventHandler`
- `pyyaml` >=6.0.2 - YAML config parsing/writing in `aws_copier/models/simple_config.py`
- `aiofiles` >=24.1.0 - Async file I/O for MD5 computation in `aws_copier/core/file_listener.py`
- `python-dotenv` >=1.1.1 - `.env` loading in the `s3_manager.py` `main()` test harness

**Infrastructure:**
- `ruff` >=0.12.11 - Listed as a runtime dependency (unusual) and also in dev; used for linting and formatting

## Configuration

**Environment:**
- AWS credentials can be loaded from environment variables: `AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`, `AWS_REGION`, `S3_BUCKET`, `S3_PREFIX`
- Primary config is a YAML file (default path `~/aws-copier-config.yaml`; dev path `config.yaml` in project root)
- `.env` files are gitignored and supported via `python-dotenv`

**Build:**
- `pyproject.toml` — single source of truth for project metadata, dependencies, tool config (ruff, mypy, black, pytest)
- `uv.lock` — locked dependency tree

## Code Quality Tools

**Linting/Formatting:**
- `ruff` >=0.12.11 — linter + formatter (replaces flake8 + isort); configured in `pyproject.toml` under `[tool.ruff]`
- `black` >=23.0.0 (dev) — formatter (line-length 88, target py39); may be superseded by ruff-format
- Pre-commit hooks in `.pre-commit-config.yaml`: `ruff`, `ruff-format`, `uv-lock`, standard file checks
- `ty` type checker runs as a pre-commit hook (local hook using `ty check`)
- `mypy` >=1.7.0 (dev) — strict static type checking configured in `pyproject.toml` under `[tool.mypy]`

**IDE:**
- VSCode settings in `.vscode/settings.json` — ruff enabled, pytest integration, uv venv path

## Platform Requirements

**Development:**
- Python 3.11, `uv` installed
- macOS, Linux, or Windows supported
- Run: `uv sync` then `uv run python main.py`

**Production:**
- Cross-platform daemon (macOS, Linux, Windows)
- Windows-specific setup script at `setup_windows.py`
- Windows batch files (`run_aws_copier.bat`, `stop_aws_copier.bat`) generated by setup script
- No containerization or deployment pipeline detected
